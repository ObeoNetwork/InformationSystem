Requirement package

Overview

This documentation describes services and attributes available on the Requirement concepts available in M2Doc generation templates.

Each concept is described in a page of it's own. Services and attributes are described using

  • a name,
  • the syntax to be used in M2Doc generation templates,
  • a description.

For some attributes and services, a usage example is provided.

Documentation of M2Doc Services for the Requirement package

Each concept of Requirement is documented in it's own page :

  • M2Doc Services and Attributes for Category
  • M2Doc Services and Attributes for EObject
  • M2Doc Services and Attributes for Repository
  • M2Doc Services and Attributes for Requirement

Viewpoints

Requirements

The Requirements viewpoint provides two representations :

  • Requirements Table : a table allowing to edit the requirements,
  • Requirements Traceability Matrix : a table tracing the relations between objects and requirements.
