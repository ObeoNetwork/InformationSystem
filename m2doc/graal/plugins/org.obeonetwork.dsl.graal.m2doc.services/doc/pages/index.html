Graal package

Overview

This documentation describes services and attributes available on the Graal concepts available in M2Doc generation templates.

Each concept is described in a page of it's own. Services and attributes are described using

  • a name,
  • the syntax to be used in M2Doc generation templates,
  • a description.

For some attributes and services, a usage example is provided.

Viewpoints

Graal Methodology

The Graal Methodology viewpoint provides ten representations :

  • Actions Plan : a diagram allowing to represent and edit the action plan of a Task,
  • Actors Graph : a diagram allowing to represent and edit the Actors of a System,
  • Actors list : a cross table showing the assignations of Actors to Tasks for a Use Case,
  • Domain Classes Diagram : a diagram allowing to represent and edit the Domain Classes of a Namespace,
  • Domain Classes Namespaces Hierarchy : a diagram allowing to represent and edit the Namespace hierarchy of a System,
  • Tasks Graph : a diagram allowing to represent and edit the Tasks (and Task Groups) of a System and assign the Actors to Tasks,
  • TasksGroup Graph : a diagram allowing to represent and edit the Tasks of a Task Group and assign the Actors to Tasks,
  • Use Case Diagram : a diagram allowing to represent and edit the Tasks of a Use Case and assign the Actors to Tasks,
  • Use Cases Main View : a diagram allowing to represent and edit the Use Cases of a System and edit relationships among them,
  • User Stories Table : a cross table showing the assignations of Tasks to User Stories for a System, allowing to create User Stories.

Requirements (Graal consolidated view)

The Requirements (Graal consolidated view) viewpoint provides two representations :

  • Requirements - Task traceability matrix : a cross table showing the requirements coverage by the Tasks,
  • Requirements - UseCase traceability matrix : a cross table showing the requirements coverage by the Use Cases.
