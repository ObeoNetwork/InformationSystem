Services and attributes for the DataBase concept

A database in the studio is a concept grouping

  • tables
  • shemas
  • sequences
  • user libraries definition.

A database has attributes and services described below.
In the following examples, the db variable points to a database.

Attributes

Name Returned type M2Doc Syntaxe Description Example
ID String {m:db.ID} returns the identifier of the database the database identifier: {m:db.id}
comments String {m:db.comments} returns the comments stored on the database comments on the database are :{m:db.comments}
techID String {m:db.techID} returns the technical id of the database
name String {m:db.name} returns the name of the database
tables List of Tables and Views {m:db.tables} returns the list of tables and views directly stored in the database (not in it's schemas)
sequences List of Sequences {m:db.sequences} returns the list of sequences directlry stored in the database (not in it's schemas)
usedLibraries List of user defined libraries {m:db.usedLibraries} returns the list of native type libraries used by the database
url String {m:db.url} returns the URL of the database
schemas List of Schemas {m:db.schemas} returns the list of schemas of the database
defines List of user defined types libraries {m:db.defines} returns the list of user defined type libraries

Services

Name Returned type M2Doc Syntaxe Description Example
tables List of Tables {m:db.tables()} returns the set of tables directly stored in the database (and not in the schemas) the list of database tables : {m:for table | db.tables()}table {m:table.name},{m:endfor}
views List of Views {m:db.views()} returns the set of views directly stored in the database (and not in the schemas) the list of database views : {m:for table | db.views()}view {m:view.name},{m:endfor}
allTables List of Tables {m:db.allTables()} returns the set of all tables including the tables in schemas and referenced through foreignkeys the list of database tables : {m:for table | db.allTables()}table {m:table.name},{m:endfor}
allSequences List of Sequences {m:db.allSequences()} returns the set of all sequences related to a table in the result of the allTable() service. the list of database sequences : {m:for sequence | db.allSequences()}sequence {m:sequence.name},{m:endfor}
allViews List of Views {m:db.allViews()} returns the set of all views contained in a database, it's schemas and the related databases (through foreign keys) the list of database views: {m:for view | db.allView()}sequence {m:view.name},{m:endfor}
localTables List of Tables {m:db.localTables()} returns the list of tables directly stored in the database and its schemas
localViews List of Views {m:db.localViews()} returns the list of views directly stored in the database and its schemas
localSequences List of Sequences {m:db.localSequences()} returns the list of sequence directly stored in the database and it's schemas
typeLibraryName String {m:db.typeLibraryName()} Returns the name of the types library used. This name contains the name of the database's engine vendor for physical models.
